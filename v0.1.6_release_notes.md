# v0.1.6 — Greedy 3-exit baseline with standardized clip-policy evaluation

## Release label
`v0.1.6`

## Suggested release title
**v0.1.6 — Greedy 3-exit baseline + standardized Depth×Time reporting**

## Summary
This release stabilizes the **v0.1 greedy 3-exit baseline** and standardizes evaluation outputs so results can be compared fairly with later variants.

## Main changes
- Added standardized **segment policy** reporting
- Added standardized **clip policy** reporting for:
  - full-clip baseline
  - Depth×Time early stopping
- Added consistent saved JSON artifacts:
  - `policy_results.json`
  - `clip_policy_results_full.json`
  - `clip_policy_results_time.json`
  - `clip_policy_results.json`
  - `windows_used_hist.json`
- Updated `summary.json` flow so later table filling is easier
- Updated documentation (`README.md`, `DOC_STRUCTURE.md`) to match the real pipeline

## Metrics now reported consistently
- Policy test accuracy
- Avg exit depth
- Exit mix
- Flip-any rate
- Exit consistency
- Clip accuracy
- Segment accuracy over processed windows
- Segment accuracy over used windows
- Avg windows used
- Windows saved
- Avg compute units
- Avg depth per used window
- Compute saved
- Fixed-position diagnostics

## Intended use
This tag should be used as the reference point for the **historical v0.1 greedy baseline** in the 3-exit comparison table.

## Scope note
This release is still **greedy-only** for the v0.1 line. It does **not** add:
- EA policy
- hint passing
- 5-exit / K-exit logic

Those belong to later versions.
